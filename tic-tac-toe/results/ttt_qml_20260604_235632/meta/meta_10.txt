# GLOBAL INSIGHTS SCRATCHPAD

The following are global insights about optimization approaches and their effectiveness:

## Successful Algorithmic Patterns  
- None of the nine evaluated programs achieved a combined score above 0.0; the initial baseline (Generation 0’s simple RX‐feature map + CZ entangler ansatz) remains tied as the “best” with zero accuracy.  
- Common elements present in every variant—class‐balanced enumeration of all legal boards and Adam optimization on L₂ loss—were necessary for reproducibility but did not on their own yield any predictive power.  
- Increasing ansatz depth via three data re-uploads and two repeats (Generations 5–8) neither improved nor degraded performance relative to shallower versions, indicating that mere repetition of the same block is insufficient for expressivity.  
- The use of a central ANSATZ_SPEC validation utility (Generation 9) enforced correct gate‐and‐wire assignments but did not translate into learning gains under the current circuit designs.  

## Ineffective Approaches  
- Fixed CZ‐only entanglement with simple RX feature maps (Gen 5 “shared-ry-across-qubits”) failed to break chance‐level performance (0.0), highlighting insufficient two-qubit parameterization.  
- Extensive parameter sharing across repeated RY/RZ layers (Gen 6 “static-spec-seed-fix”) under-expressed the variational circuit, yielding no improvement over the baseline.  
- Misaligned parameter-to-gate mappings in `apply_ansatz_block` (Gen 8 “none”) disrupted gradient flow; the resulting models never converged from their random initialization.  
- ANSATZ_SPEC validation errors (Gen 9 “fix_ansatz_spec_placeholder”) caused immediate training abortion, demonstrating that invalid gate or wire specifications can nullify an otherwise correct training pipeline.  

## Implementation Insights  
- All programs relied on RX rotations to encode each of the nine board cells, followed by per-qubit RY/RZ rotations and nearest-neighbor CZ gates—an architecture that consistently lacked the expressivity to separate three outcome classes.  
- Pure Pauli-Z expectation aggregation into class scores (without nonlinear postprocessing or richer observables) produced outputs that remained linearly inseparable across all tested circuits.  
- The validate_ansatz_spec routine effectively detected unsupported gates and bad wire indices (Gen 9), emphasizing the need for strict correspondence between the ANSATZ_SPEC definition and parameter‐key indexing in `apply_ansatz_block`.  
- No variant incorporated parameterized two-qubit rotations (e.g., CRX/CRY) or alternative entangling topologies; every attempt to boost depth via block repetition alone failed to enrich the circuit’s hypothesis space.  

## Performance Analysis  
- Across Generations 0 through 9, every program scored exactly 0.0 on combined metrics, indicating no implementation outperformed the initial baseline.  
- Structural tweaks—whether adjusting repeat counts (Gen 5 vs. Gen 7), fixing random seeds (Gen 6), or refining spec validation (Gen 9)—showed zero correlation with any improvement in validation accuracy or loss.  
- Programs aborted by spec errors (Gen 9) underscore that robustness checks are critical for functionality but must be paired with an expressive ansatz to yield meaningful results.  
- The persistent chance‐level performance across diverse circuit variants signals that future progress will require fundamentally different feature maps, richer entangling gates, or nonlinear measurement schemes rather than incremental tweaks to the existing RX–RY/RZ–CZ pattern.

# META RECOMMENDATIONS

The following are actionable recommendations for the next program generations:

1. Extend the entangling layer with parameterized two-qubit ZZ-rotations.  
   – In your ANSATZ_SPEC, add a new block entry “rzz_block” that lists each adjacent qubit pair (e.g. [[0,1],[1,2],…]).  
   – In apply_ansatz_block, apply cirq.rzz(θ) on those pairs using optimizer keys “rzz_0”, “rzz_1”,… and confirm they appear in the param list.  
   This directly addresses the global insight that tunable two-qubit rotations (RZZ) have never been tried and can enrich expressivity beyond fixed CZ gates.  

2. Interleave data re-uploads after every variational block to reinforce input encoding.  
   – After each sequence of RY/RZ/RZZ (or CZ) gates, reapply your nine-qubit RX feature map using fresh optimizer-independent operations.  
   – Update your circuit-building loop to alternate “feature_map → variational_block → feature_map → variational_block…”.  
   Prior work shows that repeated data injections break the flatness induced by a one-time encoding and can create nonlinear dependence on inputs.  

3. Move from L₂ regression to a proper three-class cross-entropy pipeline with a softmax head.  
   – After circuit execution, measure both 〈Z〉 and 〈X〉 on each qubit (yielding 18 features) and concatenate them.  
   – Feed this 18-dim vector into a small trainable softmax layer (e.g. a single Dense(3) layer) and optimize the cross-entropy loss via Adam.  
   This combats the linear inseparability of pure Z-averaging and leverages a nonlinear mapping that’s essential for classification.  

4. Vary entanglement topology per layer using reproducible random pairings.  
   – For each variational block index i, seed a pseudo-random shuffle of qubit indices and pair them in ANSATZ_SPEC under “entangle_layer_i”: [[a,b],[c,d],…].  
   – In apply_ansatz_block, read these layer-specific pairs and apply CZ or RZZ accordingly.  
   Layer-to-layer connectivity diversity has been shown to increase expressivity compared to a rigid nearest-neighbor chain.  

5. Incorporate two-qubit correlator measurements as additional features.  
   – After running the ansatz, construct circuits to measure 〈ZₖZₖ₊₁〉 on each adjacent qubit pair (you’ll get 8 extra features for 9 qubits).  
   – Append these correlators to your 〈Z〉 and 〈X〉 vectors before the softmax head.  
   This richer observable set captures entanglement patterns directly and mitigates the failure of single-qubit expectation summaries to separate the classes.
