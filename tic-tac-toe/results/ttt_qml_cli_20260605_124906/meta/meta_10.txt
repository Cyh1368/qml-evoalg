# GLOBAL INSIGHTS SCRATCHPAD

The following are global insights about optimization approaches and their effectiveness:

## Successful Algorithmic Patterns  
- Static-perimeter CZ + parameterized-hub CRZ  
  • The current best program (Combined 0.57, train 0.54 → val 0.60 → test 0.48, 60 params) replaces the Gen 6 perimeter CRZ gates (66 params, Combined 0.53) with fixed CZ on the ring and leaves only CRZ on the hub.  
  • This change cut 6 parameters (from 66 to 60) while boosting combined score by 0.04, showing that parameter sharing on the ring sufficed for expressivity.  
- Five-layer symmetric rotations  
  • All top scorers (Gen 1 best, this program) use shared RZ “pre-” rotations, shared RY rotations, then shared “post-” RZ rotations across the corner/edge/center groups.  
  • This hierarchy (pre-entangle → rotate → entangle → rotate → post-entangle) consistently yielded combined scores ≥ 0.57.  
- Fixed 3 uploads × 2 repeats data-reupload backbone  
  • Retaining N_UPLOADS=3 and N_REPEATS=2 (depth ~70, gate count ~261) provided a stable temporal expressivity scaffold across generations.  
- Grouped Z-expectation readouts by symmetry for outputs  
  • Mapping the four edge qubits to “cross,” four corner qubits to “circle,” and the center qubit to “draw” compressed nine observables into three class logits with minimal overhead and robust classification.  

## Ineffective Approaches  
- Parameterized ring entanglement  
  • Gen 6’s use of CRZ around the perimeter (66 params) yielded only Combined 0.53 versus 0.57 with static CZ. Introducing perimetric parameters added capacity but not usable expressivity.  
- Mixed CRX/CRY hub couplings  
  • Gen 9’s hybrid CRX/CRY on the hub (78 params) achieved Combined 0.55 and test accuracy 0.50 but dropped parameter efficiency to 0.006, indicating diminishing returns from extra two-qubit parameter types.  
- Excessive circuit depth/gate count without structural hierarchy  
  • Gen 7/8 circuits extended depth to 76–84 and gate counts to 285 by adding CNOTs or extra rotations but fell to Combined 0.48–0.45, showing that depth alone cannot compensate for structured entanglement.  
- Invalid diagonal entanglement  
  • Gen 5’s inclusion of CZ on diagonals (0–4, 2–6) failed the adjacency check and produced a zero score. Any non-adjacent gates must be added to GRID_EDGE_SET or removed to pass spec validation.  

## Implementation Insights  
- Centralized ANSATZ_SPEC builder  
  • Encapsulating the entire ansatz in `_build_hierarchical_ansatz_spec()` makes swapping entanglement types or symmetry groupings a one-place edit. E.g., changing perimeter CRZ→CZ merely alters two lines in the spec.  
- Spec validation via GRID_EDGE_SET  
  • `validate_ansatz_spec` uses the canonical `GRID_EDGE_SET` to catch illegal two-qubit gates early. Expanding `GRID_EDGE_SET` to include new edges (e.g., diagonals) or correcting specs prevents wasted training.  
- Param-key mapping for efficient indexing  
  • Generating `PARAMETER_KEYS_PER_BLOCK` via an ordered dict ensures consistent parameter ordering across blocks, simplifying `apply_ansatz_block` to a stateless loop over `ANSATZ_SPEC`.  
- Queued circuit metadata collection  
  • Wrapping feature maps and ansatz blocks in an `AnnotatedQueue` and extracting `tape.graph.get_depth()` yields reproducible depth/gate counts. This tight feedback loop correlated design tweaks directly to depth and gate metrics.  

## Performance Analysis  
- Parameter count vs combined score  
  • 60 params → Combined 0.57 (best)  
  • 66 params → Combined 0.53 (Gen 6)  
  • 78 params → Combined 0.55 (Gen 9)  
  • 90 params → Combined 0.45 (Gen 8)  
  This clear peak at 60 params highlights an optimal expressivity/efficiency sweet spot.  
- Depth/gate count sweet spot around (70, 261)  
  • Best program and Gen 9 share depth 70, gate 261 with top combined scores (0.57 and 0.55), whereas deeper variants (76–84, 285 gates) underperform.  
- Generalization gap and stability  
  • Best generalization gap 0.06 (0.54→0.48) with zero train/val std 0.00, confirming stable convergence under Adam + balanced L₂ loss.  
  • Underfitting variants saw gaps up to 0.10 (Gen 8) or marginal overfitting with little expressivity.  
- Parameter efficiency peaks at ~0.008  
  • Best program’s efficiency (test 0.48/60 params ≈ 0.008) surpasses Gen 9 (0.50/78 ≈ 0.006) and Gen 6 (0.45/66 ≈ 0.007), underscoring that fewer, well-placed parameters can outperform larger mixed-gate ansätze.

# META RECOMMENDATIONS

The following are actionable recommendations for the next program generations:

1. Introduce an intermediate phase‐alignment rotation between the ring and hub layers. In `_build_hierarchical_ansatz_spec`, after the eight CZ perimeter gates (Layer 3), insert a new “Layer 3.5” of RZ rotations on corners, edges, and center using three new shared keys (e.g. `rz_mid_corner`, `rz_mid_edge`, `rz_mid_center`). This mirrors the proven pre-/post-entanglement blocks and can better condition the ring outputs before the adaptive CRZ hub couplings.  

2. Test a lightly parameterized ring by alternating CZ and CRZ on the perimeter. Replace four of the eight static CZs—say edges (0,1),(2,3),(4,5),(6,7)—with a single shared‐parameter CRZ (keyed `crz_ring_alt`), leaving the alternate edges as CZ. This adds only one extra parameter and evaluates whether partial ring parameterization boosts expressivity without over-parameterizing the entire ring.  

3. Add a symmetry-preserving diagonal entanglement layer. After the hub CRZ gates, append two CRZ gates between qubits (0,4) and (2,6) with a shared key `crz_diag`, and extend `GRID_EDGE_SET` to include these diagonal pairs. This introduces non-local couplings that respect the board’s four-fold symmetry and may capture cross-board correlations missed by purely local entanglement.  

4. Diversify the hub entanglement axes within the same parameter budget. In the hub layer, alternate half the center-to-edge gates to CRX (e.g. use `crx_hub_edge` on edges (8,3) and (8,7) and keep `crz_hub_edge` on (8,1),(8,5)), sharing one parameter per axis. This hybrid CRZ/CRX pattern preserves the total parameter count while exploring richer two-qubit interactions at the central node.  

5. Consolidate pre- and post-entanglement RZ keys to deepen parameter sharing. Merge `rz_pre_corner`/`rz_post_corner` into a single `rz_corner` (and similarly for edges/center) in the spec, halving your RZ parameters per block. This tighter sharing tests whether unified phase rotations across both blocks can maintain the 0.57 combined score while improving parameter efficiency further.
