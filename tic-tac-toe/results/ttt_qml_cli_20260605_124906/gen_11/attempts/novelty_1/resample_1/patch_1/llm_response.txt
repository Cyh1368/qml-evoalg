Inspired by prior attempts to diversify hub entangling axes while keeping a compact parameter budget, this block adopts global (shared) ring entanglement via CRX around the board perimeter and a shared hub entanglement via CRY from the center to each edge. This reduces parameter count per block (11 keys vs 13 in the previous variant) and leverages structured symmetry: shared pre/post-RZ and shared RY layers preserve Tic-Tac-Toe board symmetries while the ring/hub interactions provide richer inter-qubit correlations without exploding the parameter budget. This mirrors the second program’s improvements (0.60 combined score) by collapsing ring and hub gates to single shared parameters, potentially improving generalization and convergence with Adam.

def _build_hierarchical_ansatz_spec() -> list[dict]:
    """Build hierarchical entanglement ansatz with parameter sharing.

    Structural changes:
    1) Pre-entanglement RZ on all qubits (shared by symmetry)
    2) RY rotations on all qubits (shared by symmetry)
    3) Ring entanglement: CRX gates around outer perimeter (shared angle)
    4) Hub entanglement: CRY gates center-to-edges (shared angle)
    5) Post-entanglement RZ on all qubits (shared by symmetry)
    """
    spec: list[dict] = []

    # Symmetry groups for 4-fold rotational symmetry of tic-tac-toe board
    # Corners: 0, 2, 4, 6 (all equivalent by rotation)
    # Edges: 1, 3, 5, 7 (all equivalent by rotation)
    # Center: 8 (unique)

    # Layer 1: Pre-entanglement RZ rotations with parameter sharing
    spec.append({"gate": "RZ", "wire": 0, "param": "rz_pre_corner"})
    spec.append({"gate": "RZ", "wire": 2, "param": "rz_pre_corner"})

    spec.append({"gate": "CRY", "wires": [CENTER, 1], "param": "cry_hub_edge"})
